# Requirements Document: AI Chat Widget

## Introduction

The AI Chat Widget is an intelligent, AI-powered chatbot component that provides 24/7 customer service and lead capture capabilities for TomNerb Digital Solutions. The widget operates in two modes: AI Customer Service Mode (default) for answering questions about TomNerb's services, and Lead Capture Mode for collecting contact information from potential clients. The widget is persistent across all pages, mobile-responsive, and designed to be a key differentiator in the Cambodian market where live support is uncommon.

## Glossary

- **Widget**: The AI Chat Widget component
- **Chat_Panel**: The expanded chat interface (350px × 500px)
- **Collapsed_Button**: The circular button shown when the widget is minimized
- **AI_Service**: The Anthropic Claude API integration (claude-haiku-4-5)
- **Lead_Capture_System**: The email-based lead collection subsystem
- **Session**: A single browser session from page load to close
- **Quick_Reply_Chip**: A clickable button suggesting common questions
- **Typing_Indicator**: Visual feedback showing the AI is processing
- **Email_Service**: The Resend SDK integration for sending emails
- **User_Message**: A message sent by the website visitor
- **AI_Message**: A response generated by the AI_Service
- **System_Prompt**: The instruction set that defines the AI's behavior and knowledge
- **Rate_Limiter**: A mechanism to prevent API abuse

## Requirements

### Requirement 1: Widget Visibility and Positioning

**User Story:** As a website visitor, I want to see the chat widget on every page, so that I can get help whenever I need it.

#### Acceptance Criteria

1. THE Widget SHALL be positioned in the bottom-right corner of the viewport
2. THE Widget SHALL remain visible across all pages of the website
3. THE Widget SHALL maintain its position when the user scrolls
4. THE Widget SHALL be rendered as a global component in the application layout
5. WHEN the viewport width is less than 768px, THE Widget SHALL adapt its dimensions to fit mobile screens

### Requirement 2: Collapsed State Display

**User Story:** As a website visitor, I want to see an attractive button that invites interaction, so that I know help is available.

#### Acceptance Criteria

1. WHEN the Widget is collapsed, THE Collapsed_Button SHALL display a circular button with a sparkle icon (✦)
2. WHILE the Widget is collapsed, THE Collapsed_Button SHALL display a continuous pulse ring animation
3. THE Collapsed_Button SHALL have a minimum touch target size of 56px × 56px for accessibility
4. WHEN the user hovers over the Collapsed_Button, THE Widget SHALL display a visual hover state
5. THE Collapsed_Button SHALL use the accent color (#4F7CFF) as its primary color

### Requirement 3: Widget Expansion and Collapse

**User Story:** As a website visitor, I want to open and close the chat smoothly, so that I have a pleasant user experience.

#### Acceptance Criteria

1. WHEN the user clicks the Collapsed_Button, THE Widget SHALL expand to display the Chat_Panel
2. WHEN the Widget expands, THE Chat_Panel SHALL animate smoothly using Framer Motion
3. WHEN the user clicks the close button in the Chat_Panel, THE Widget SHALL collapse to the Collapsed_Button state
4. WHEN the Widget collapses, THE Chat_Panel SHALL animate smoothly out of view
5. THE Widget SHALL expand and collapse without causing a page refresh

### Requirement 4: Chat Panel Dimensions and Layout

**User Story:** As a website visitor, I want a well-sized chat interface, so that I can read messages comfortably.

#### Acceptance Criteria

1. WHEN the Widget is expanded on desktop, THE Chat_Panel SHALL be 350px wide and 500px tall
2. THE Chat_Panel SHALL include a header section with the TomNerb branding and close button
3. THE Chat_Panel SHALL include a scrollable message area
4. THE Chat_Panel SHALL include an input area at the bottom for user messages
5. WHEN the viewport width is less than 768px, THE Chat_Panel SHALL occupy 100% of the viewport width minus appropriate margins

### Requirement 5: AI Customer Service Mode Initialization

**User Story:** As a website visitor, I want the chat to start in AI mode by default, so that I can immediately ask questions.

#### Acceptance Criteria

1. WHEN the Widget is first opened, THE Widget SHALL initialize in AI Customer Service Mode
2. THE Widget SHALL display a welcome message from the AI
3. THE Widget SHALL display Quick_Reply_Chips for common questions
4. THE System_Prompt SHALL define the AI as "TomNerb's friendly digital assistant"
5. THE System_Prompt SHALL include information that TomNerb Digital Solutions is a Cambodian startup helping SMEs with custom software, automation, and business management systems

### Requirement 6: Quick Reply Chips

**User Story:** As a website visitor, I want to see suggested questions, so that I can quickly get the information I need.

#### Acceptance Criteria

1. WHEN the Widget is in AI Customer Service Mode, THE Widget SHALL display Quick_Reply_Chips below the welcome message
2. WHEN the user clicks a Quick_Reply_Chip, THE Widget SHALL send that question as a User_Message
3. THE Quick_Reply_Chips SHALL include options such as "What services do you offer?", "How much does it cost?", "Tell me about your team"
4. WHEN a Quick_Reply_Chip is clicked, THE Widget SHALL remove the chips from view
5. THE Quick_Reply_Chips SHALL be visually distinct from regular messages

### Requirement 7: User Message Input

**User Story:** As a website visitor, I want to type and send messages, so that I can ask custom questions.

#### Acceptance Criteria

1. THE Chat_Panel SHALL provide a text input field for User_Messages
2. WHEN the user types in the input field, THE Widget SHALL enable the send button
3. WHEN the user presses Enter in the input field, THE Widget SHALL send the User_Message
4. WHEN the user clicks the send button, THE Widget SHALL send the User_Message
5. WHEN a User_Message is sent, THE Widget SHALL clear the input field

### Requirement 8: Message Display

**User Story:** As a website visitor, I want to see my messages and AI responses clearly, so that I can follow the conversation.

#### Acceptance Criteria

1. WHEN a User_Message is sent, THE Widget SHALL display it in the message area with user-specific styling
2. WHEN an AI_Message is received, THE Widget SHALL display it in the message area with AI-specific styling
3. THE Widget SHALL display User_Messages aligned to the right
4. THE Widget SHALL display AI_Messages aligned to the left
5. THE Widget SHALL automatically scroll to the latest message when a new message is added

### Requirement 9: AI Response Generation

**User Story:** As a website visitor, I want the AI to answer my questions accurately, so that I can learn about TomNerb's services.

#### Acceptance Criteria

1. WHEN a User_Message is sent in AI Customer Service Mode, THE Widget SHALL send the message to the AI_Service
2. THE AI_Service SHALL use the claude-haiku-4-5 model
3. THE AI_Service SHALL include the System_Prompt in every API request
4. WHEN the AI_Service returns a response, THE Widget SHALL display it as an AI_Message
5. THE AI_Service SHALL respond in a warm and professional tone as defined in the System_Prompt

### Requirement 10: Typing Indicator

**User Story:** As a website visitor, I want to see when the AI is thinking, so that I know my message was received.

#### Acceptance Criteria

1. WHEN a User_Message is sent to the AI_Service, THE Widget SHALL display the Typing_Indicator
2. THE Typing_Indicator SHALL show animated dots
3. WHEN the AI_Service returns a response, THE Widget SHALL remove the Typing_Indicator
4. THE Typing_Indicator SHALL be visually distinct and positioned where the AI_Message will appear
5. THE Typing_Indicator animation SHALL loop continuously until the response is received

### Requirement 11: Escape Hatch to Lead Capture

**User Story:** As a website visitor, I want to connect with the team directly if the AI can't help, so that I can get personalized assistance.

#### Acceptance Criteria

1. WHEN the AI cannot answer a question, THE Widget SHALL respond with "Let me connect you with our team directly — what's your email?"
2. WHEN the AI provides the escape hatch message, THE Widget SHALL switch to Lead Capture Mode
3. THE Widget SHALL maintain the conversation history when switching modes
4. THE Widget SHALL clearly indicate the mode transition to the user
5. THE escape hatch SHALL be triggered by AI discretion based on question complexity or knowledge gaps

### Requirement 12: Manual Lead Capture Trigger

**User Story:** As a website visitor, I want to request a quote or talk to the team directly, so that I can get personalized service.

#### Acceptance Criteria

1. WHEN the user clicks a "Talk to the Team" Quick_Reply_Chip, THE Widget SHALL switch to Lead Capture Mode
2. WHEN the user clicks a "Get a Quote" Quick_Reply_Chip, THE Widget SHALL switch to Lead Capture Mode
3. THE Widget SHALL display a message explaining that it will collect contact information
4. THE Widget SHALL maintain the conversation history when switching modes
5. THE mode switch SHALL occur without page refresh or widget collapse

### Requirement 13: Lead Information Collection

**User Story:** As a website visitor, I want to provide my contact information easily, so that the team can reach out to me.

#### Acceptance Criteria

1. WHEN the Widget is in Lead Capture Mode, THE Lead_Capture_System SHALL prompt for the user's name
2. WHEN the user provides their name, THE Lead_Capture_System SHALL prompt for the user's email
3. WHEN the user provides their email, THE Lead_Capture_System SHALL prompt for their challenge or message
4. THE Lead_Capture_System SHALL validate that the email follows a valid email format
5. THE Lead_Capture_System SHALL collect all three fields (name, email, message) before submission

### Requirement 14: Lead Submission

**User Story:** As a website visitor, I want my information sent to TomNerb, so that they can follow up with me.

#### Acceptance Criteria

1. WHEN all lead information is collected, THE Lead_Capture_System SHALL send the data to info@tomnerb.com via the Email_Service
2. THE Email_Service SHALL use the Resend SDK
3. THE email SHALL include the user's name, email, and message in a formatted structure
4. WHEN the email is successfully sent, THE Widget SHALL display a success confirmation message
5. THE success message SHALL thank the user and indicate that the team will respond soon

### Requirement 15: Session Persistence

**User Story:** As a website visitor, I want my chat history to remain when I navigate between pages, so that I don't lose context.

#### Acceptance Criteria

1. WHEN the user navigates to a different page, THE Widget SHALL maintain the message history
2. THE Widget SHALL store the message history in session storage
3. WHEN the user closes and reopens the Widget during the same Session, THE Widget SHALL restore the message history
4. WHEN the user closes the browser or tab, THE Widget SHALL clear the message history
5. THE Widget SHALL maintain the current mode (AI or Lead Capture) across page navigations during the same Session

### Requirement 16: API Error Handling

**User Story:** As a website visitor, I want to see helpful error messages if something goes wrong, so that I understand what happened.

#### Acceptance Criteria

1. WHEN the AI_Service fails to respond, THE Widget SHALL display an error message to the user
2. WHEN the Email_Service fails to send, THE Widget SHALL display an error message to the user
3. THE error messages SHALL be user-friendly and avoid technical jargon
4. WHEN an API error occurs, THE Widget SHALL suggest alternative actions (e.g., "Try again" or "Email us directly at info@tomnerb.com")
5. THE Widget SHALL log API errors to the browser console for debugging purposes

### Requirement 17: Rate Limiting

**User Story:** As the system owner, I want to prevent API abuse, so that costs remain controlled and service remains available.

#### Acceptance Criteria

1. THE Rate_Limiter SHALL limit the number of AI_Service requests per Session
2. THE Rate_Limiter SHALL limit the number of Email_Service requests per Session
3. WHEN the rate limit is exceeded, THE Widget SHALL display a message asking the user to wait before sending more messages
4. THE Rate_Limiter SHALL allow a maximum of 20 AI messages per Session
5. THE Rate_Limiter SHALL allow a maximum of 3 lead submissions per Session

### Requirement 18: Lazy Loading

**User Story:** As a website visitor, I want the page to load quickly, so that I can access content immediately.

#### Acceptance Criteria

1. THE Widget SHALL be lazy-loaded after the initial page content
2. THE Widget SHALL not block the rendering of critical page content
3. THE Widget's JavaScript bundle SHALL be loaded asynchronously
4. WHEN the page loads, THE Widget SHALL have zero impact on the initial page load performance metrics
5. THE Collapsed_Button SHALL appear within 2 seconds of page load completion

### Requirement 19: Keyboard Navigation

**User Story:** As a keyboard user, I want to navigate the chat using my keyboard, so that I can use the widget without a mouse.

#### Acceptance Criteria

1. THE Collapsed_Button SHALL be focusable using the Tab key
2. WHEN the Collapsed_Button is focused, pressing Enter or Space SHALL expand the Widget
3. THE Chat_Panel input field SHALL be focusable using the Tab key
4. THE close button SHALL be focusable and operable using the keyboard
5. WHEN the Chat_Panel is open, pressing Escape SHALL collapse the Widget

### Requirement 20: Screen Reader Support

**User Story:** As a screen reader user, I want to understand the chat interface, so that I can interact with the widget effectively.

#### Acceptance Criteria

1. THE Collapsed_Button SHALL have an aria-label describing its purpose
2. THE Chat_Panel SHALL have appropriate ARIA roles and labels
3. WHEN a new AI_Message is received, THE Widget SHALL announce it to screen readers using aria-live
4. THE input field SHALL have a descriptive label for screen readers
5. THE Widget SHALL follow WCAG 2.1 Level AA accessibility guidelines

### Requirement 21: Visual Theme

**User Story:** As a website visitor, I want the chat to match the site's design, so that it feels integrated and professional.

#### Acceptance Criteria

1. THE Widget SHALL use a dark theme with background color #060810
2. THE Widget SHALL use accent color #4F7CFF for interactive elements
3. THE Chat_Panel SHALL have rounded corners and subtle shadows for depth
4. THE message bubbles SHALL have distinct colors for User_Messages and AI_Messages
5. THE Widget SHALL use the same typography as the main website

### Requirement 22: Animation Performance

**User Story:** As a website visitor, I want smooth animations, so that the interface feels polished and responsive.

#### Acceptance Criteria

1. THE Widget SHALL use Framer Motion for all animations
2. THE Collapsed_Button pulse animation SHALL run at 60 frames per second
3. THE Chat_Panel expand/collapse animation SHALL complete within 300 milliseconds
4. THE Typing_Indicator animation SHALL run smoothly without frame drops
5. THE Widget animations SHALL use GPU-accelerated CSS properties (transform, opacity)

### Requirement 23: Mobile Responsiveness

**User Story:** As a mobile user, I want the chat to work well on my phone, so that I can get help on any device.

#### Acceptance Criteria

1. WHEN the viewport width is less than 768px, THE Chat_Panel SHALL adjust to a mobile-optimized layout
2. WHEN on mobile, THE Chat_Panel SHALL occupy the full width minus 16px margins on each side
3. WHEN on mobile, THE Collapsed_Button SHALL remain easily tappable with a 56px × 56px touch target
4. THE Widget SHALL support touch gestures for scrolling through messages
5. WHEN the mobile keyboard appears, THE Chat_Panel SHALL adjust its height to remain visible

### Requirement 24: Configuration and Environment Variables

**User Story:** As a developer, I want to configure API keys securely, so that credentials are not exposed in the codebase.

#### Acceptance Criteria

1. THE Widget SHALL read the Anthropic API key from environment variables
2. THE Widget SHALL read the Resend API key from environment variables
3. THE Widget SHALL read the recipient email address from environment variables
4. THE environment variables SHALL not be committed to version control
5. THE Widget SHALL fail gracefully with a clear error message if required environment variables are missing

### Requirement 25: Parser for Chat Messages

**User Story:** As a developer, I want to parse and format chat messages correctly, so that special characters and formatting are handled properly.

#### Acceptance Criteria

1. THE Widget SHALL parse User_Messages to escape HTML special characters
2. THE Widget SHALL parse AI_Messages to render line breaks correctly
3. THE Widget SHALL support basic markdown formatting in AI_Messages (bold, italic, links)
4. WHEN a message contains a URL, THE Widget SHALL render it as a clickable link
5. FOR ALL valid message objects, serializing then deserializing SHALL produce an equivalent object (round-trip property)

### Requirement 26: Message Timestamp Display

**User Story:** As a website visitor, I want to see when messages were sent, so that I can track the conversation timeline.

#### Acceptance Criteria

1. THE Widget SHALL record a timestamp for each User_Message and AI_Message
2. THE Widget SHALL display timestamps in a human-readable format (e.g., "2:30 PM")
3. THE timestamps SHALL be displayed in the user's local timezone
4. THE timestamps SHALL be visually subtle and not distract from message content
5. WHEN messages are sent within 1 minute of each other, THE Widget SHALL group them and show only one timestamp

### Requirement 27: Connection Status Indicator

**User Story:** As a website visitor, I want to know if the chat is connected, so that I understand if my messages will be delivered.

#### Acceptance Criteria

1. WHEN the Widget successfully connects to the AI_Service, THE Widget SHALL display a connected status indicator
2. WHEN the Widget loses connection to the AI_Service, THE Widget SHALL display a disconnected status indicator
3. THE status indicator SHALL be visually subtle and positioned in the Chat_Panel header
4. WHEN disconnected, THE Widget SHALL attempt to reconnect automatically
5. WHEN reconnection succeeds, THE Widget SHALL update the status indicator to connected

### Requirement 28: Analytics Event Tracking

**User Story:** As a business owner, I want to track widget usage, so that I can measure its effectiveness.

#### Acceptance Criteria

1. WHEN the Widget is opened, THE Widget SHALL emit an analytics event
2. WHEN a User_Message is sent, THE Widget SHALL emit an analytics event
3. WHEN Lead Capture Mode is activated, THE Widget SHALL emit an analytics event
4. WHEN a lead is successfully submitted, THE Widget SHALL emit an analytics event
5. THE analytics events SHALL include relevant metadata (timestamp, mode, message count)

### Requirement 29: Content Security Policy Compliance

**User Story:** As a security-conscious developer, I want the widget to comply with CSP, so that the site remains secure.

#### Acceptance Criteria

1. THE Widget SHALL not use inline JavaScript that violates Content Security Policy
2. THE Widget SHALL not use inline styles that violate Content Security Policy
3. THE Widget SHALL load all external resources (APIs) over HTTPS
4. THE Widget SHALL include appropriate nonce or hash values for any required inline scripts
5. THE Widget SHALL function correctly when strict CSP headers are enabled

### Requirement 30: Graceful Degradation

**User Story:** As a website visitor with JavaScript disabled, I want to see a fallback option, so that I can still contact TomNerb.

#### Acceptance Criteria

1. WHEN JavaScript is disabled, THE Widget SHALL display a static link to the contact page
2. THE fallback link SHALL be styled consistently with the site design
3. THE fallback link SHALL be positioned in the same location as the Collapsed_Button
4. THE fallback link SHALL have descriptive text (e.g., "Contact Us")
5. THE Widget SHALL detect JavaScript availability and render the appropriate interface
